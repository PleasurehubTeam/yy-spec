# 分步工作流详解

> 30 秒了解：分步工作流让你在每个阶段暂停、审核、修改，适合需求复杂或需要精细控制的场景。

## 完整流程

以下所有命令都在 Claude Code 中以 slash command 形式执行。

### 1. 需求生成

```
/yy:spec-requirements my-feature
```

- 根据描述生成 EARS 格式的需求文档
- 自动扫描歧义并标注
- 生成后审核，满意后继续

### 2. 技术设计

```
/yy:spec-design my-feature -y
```

- 基于需求生成架构设计
- 包含组件定义、接口契约、技术选型
- `-y` 自动确认需求已审核

### 3. 任务分解

```
/yy:spec-tasks my-feature -y
```

- 将设计拆分为可执行的实现任务
- 每个任务 1-3 小时，包含验收标准
- 标识可并行执行的任务

### 4. 实现

```
/yy:spec-impl my-feature 1.1
```

- 按任务编号逐个实现
- 每个任务遵循 TDD（先写测试，再写代码）
- 建议每个任务在新的对话中执行

## 查看进度

```
/yy:status my-feature
```
