# 新项目：调查问题

> 30 秒了解：不确定问题原因时，用系统化的调查流程找到根因，调查结束后可无缝转入修复。

## 使用方式

在 Claude Code 中输入：

```
/yy:investigate "用户反馈页面偶尔加载很慢"
```

## AI 会做什么

1. **创建调查规格** — 建立 investigation 类型的规格目录
2. **收集线索** — 分析代码、日志、配置等相关信息
3. **形成假设** — 列出可能的原因和验证方法
4. **逐一验证** — 通过代码分析和测试排除假设
5. **得出结论** — 确认根因和影响范围
6. **转入修复** — 如需修复，自动转为 `/yy:fix` 工作流

## 适用场景

- 间歇性问题（偶尔出现）
- 性能问题（不确定瓶颈在哪）
- 奇怪的行为（不符合预期但不确定是不是 Bug）
- 第三方依赖问题（不确定是自己代码还是外部原因）
