# 新项目：修复 Bug

> 30 秒了解：已知 Bug 的标准修复路径 — TDD 驱动修复，自动生成测试、修复代码并更新规格。

## 使用方式

在 Claude Code 中输入：

```
/yy:fix "登录页面输入密码后无响应"
```

## AI 会做什么

1. **创建规格** — 自动建立 bugfix 类型的规格目录
2. **分析问题** — 定位 Bug 原因和影响范围
3. **编写测试** — 先写一个能复现 Bug 的失败测试
4. **修复代码** — 用最小改动修复问题，确保测试通过
5. **代码审查** — 自动审查修复质量
6. **更新记录** — 记录修复内容到变更日志

## fix vs investigate？

| 情况 | 用什么 |
|------|--------|
| 知道问题是什么，能描述症状 | `/yy:fix` |
| 不确定问题原因，需要排查 | `/yy:investigate` |
| investigate 确认原因后 | 自动转为 fix |
