# skill-creator

> 让 AI 帮你创建自己的 skill — 把你的重复性工作流程、团队规范、领域知识打包成可复用的能力模块。

## 这是什么？

skill-creator 是一个教 AI 怎么创建 skill 的 skill（有点套娃）。安装后，你可以让 AI 帮你把任何重复性工作提炼成一个标准化的 skill 包，以后再遇到同类工作，AI 就能按照你定义的流程来做。

## 安装

```bash
npx skills add anthropic/skill-creator -g -y
```

## 怎么触发？

你需要明确告诉 AI 你要创建 skill：

- "帮我创建一个 skill"
- "我想把 XX 工作流程做成一个 skill"
- "用 skill-creator 帮我打包这个流程"

## 为什么要自己创建 Skill？

### 场景 1：重复性强的工作任务

你发现自己反复跟 AI 说同样的话：

- "帮我按照公司规范写提交信息"
- "生成周报，格式跟上次一样"
- "部署到 staging，步骤跟之前说的一样"

每次都要重新解释一遍格式、步骤、注意事项。做成 skill 后，AI 直接按规范执行，你只需要说一句"帮我写周报"。

### 场景 2：团队规范和约定

你的团队有一些不成文的规矩：

- API 返回格式必须遵循某个 schema
- 代码审查要检查哪些点
- 数据库迁移必须按特定步骤执行
- 新人入职需要配置哪些环境

这些知识散落在 wiki、聊天记录、口头传授里。做成 skill 后，AI 就是你的团队知识库，任何人问都能得到一致的答案。

### 场景 3：领域专业知识

你在某个领域有专业经验：

- 金融数据查询要查哪些表、字段含义是什么
- 医疗文档有特定的术语和格式要求
- 法律合同审查需要检查哪些条款

这些知识 AI 不一定有。做成 skill 后，相当于给 AI 做了一次领域培训。

### 场景 4：复杂的多步骤流程

有些工作步骤多、容易出错：

- 发布流程：改版本号 → 更新 changelog → 打 tag → 构建 → 发布 → 通知
- 数据处理：下载 → 清洗 → 转换 → 验证 → 导入
- 环境搭建：安装依赖 → 配置环境变量 → 初始化数据库 → 启动服务

做成 skill 后，AI 按照你定义好的步骤逐一执行，不会漏步骤。

### 场景 5：带脚本的自动化任务

有些操作每次都需要写同样的代码：

- PDF 表单填写
- 图片批量处理
- 数据格式转换
- 报告生成

做成 skill 后，脚本打包在里面，AI 直接调用，不用每次重新写。

## 一个 Skill 长什么样？

```
my-skill/
├── SKILL.md              # 主文件（必须）— 告诉 AI 怎么做
├── scripts/              # 脚本（可选）— 可直接执行的代码
├── references/           # 参考资料（可选）— AI 需要查阅的知识
└── assets/               # 资源文件（可选）— 模板、图片等
```

**SKILL.md** 是核心，包含两部分：
- **头部**（YAML）：skill 的名字和触发描述 — AI 靠这个判断什么时候该用这个 skill
- **正文**（Markdown）：具体的操作指南 — AI 被触发后按这个执行

## 创建流程

告诉 AI 你想创建什么 skill，它会引导你走完整个流程：

### 1. 聊清楚需求

AI 会问你几个问题：
- 这个 skill 要解决什么问题？
- 能举几个具体的使用场景吗？
- 有没有固定的步骤或格式？

### 2. 规划内容

AI 会根据你的场景分析：
- 哪些操作适合做成脚本（每次都一样的代码）
- 哪些知识适合做成参考资料（表结构、规范文档）
- 哪些资源需要打包进去（模板、图片）

### 3. 生成 Skill

AI 会调用 `init_skill.py` 创建 skill 目录，然后：
- 写 SKILL.md
- 创建必要的脚本
- 整理参考资料
- 测试脚本能不能跑通

### 4. 打包分发

```bash
# AI 会调用打包脚本
scripts/package_skill.py my-skill/
```

生成 `.skill` 文件，可以分享给团队其他人。

### 5. 迭代改进

用了几次之后发现有些地方不够好？直接告诉 AI 改就行，它会更新 SKILL.md 和相关资源。

## 实际例子

### 例 1：周报生成 Skill

```
你：帮我做一个写周报的 skill，格式是这样的：
    标题用日期，然后分"本周完成""下周计划""风险与阻塞"三个部分，
    每个部分用列表，每条要带 JIRA 编号。

AI：（问几个问题后生成 skill）
```

以后你只需要说"帮我写周报"，AI 就按这个格式来。

### 例 2：数据库查询 Skill

```
你：我们的 BigQuery 表结构很复杂，每次问 AI 查数据都要重新解释表关系。
    帮我把表结构和常用查询做成 skill。

AI：（引导你提供表结构，生成 references/schema.md，
    写 SKILL.md 指导如何查询）
```

以后你说"这个月活跃用户有多少"，AI 知道去查哪张表、用什么字段。

### 例 3：发布流程 Skill

```
你：我们的发布流程有 8 步，每次都怕漏。帮我做成 skill。

AI：（引导你列出步骤，把可自动化的做成脚本，
    写 SKILL.md 串起来整个流程）
```

以后你说"发布 1.2.0 版本"，AI 按步骤逐一执行，不会漏。

## 写好 SKILL.md 的关键

### 触发描述要具体

AI 是通过 SKILL.md 头部的 `description` 来判断什么时候该用这个 skill 的。写得越具体，触发越准。

```yaml
# 不好 — 太模糊
description: 帮助处理文档

# 好 — 具体场景
description: >
  Use when creating, editing, or analyzing .docx files.
  Triggers on: document creation, tracked changes,
  comment insertion, format conversion.
```

### 正文要简洁

AI 本身已经很聪明了，不需要解释基础概念。只写 AI 不知道的东西：
- 你的团队规范
- 你的表结构
- 你的特定步骤
- 你的格式要求

### 大文件拆出去

如果参考资料很长（比如数据库 schema 有上百行），放到 `references/` 目录里，SKILL.md 里只写一句"查询表结构见 references/schema.md"。AI 需要时才会加载，不浪费上下文。
