# Superpowers

> 一套完整的 AI 编码工作流框架 — 让 AI 不再"随便写写"，而是遵循严格的软件工程纪律来开发。

## 这是什么？

Superpowers 是目前最受欢迎的 Claude Code 插件之一（GitHub 72k+ stars）。它是一整套**开发工作流规范**，由 15+ 个 skill 组成。

简单来说：**装了 Superpowers 之后，AI 会变得更靠谱。**

没装之前，AI 可能会：
- 想到什么写什么，没有计划
- 跳过测试，直接写代码
- 遇到 bug 乱猜，反复试错
- 写完就说"搞定了"，实际上没验证过

装了之后，AI 会被强制要求：
- 先头脑风暴，再做设计，最后才写代码
- 严格遵循 TDD（先写测试，再写实现）
- 遇到 bug 按科学方法排查，不许乱猜
- 完成前必须拿出验证证据，不许说"应该没问题"

## 装了之后会怎样？

### 你说"我想做个 XX 功能"

AI 不会直接写代码。它会：

1. **先问你问题** — 每次只问一个，搞清楚你到底要什么
2. **提出 2-3 种方案** — 带优缺点和推荐意见
3. **分段展示设计** — 每部分都征求你的意见
4. **写设计文档** — 保存下来，然后才开始写代码

哪怕你觉得"就改一行代码"，AI 也会先确认再动手。这是故意的 — "简单"任务往往是翻车最多的地方。

### AI 开始写代码时

严格遵循 TDD 循环：

1. **先写一个会失败的测试**（证明测试有效）
2. **写最少的代码让测试通过**（不多写一行）
3. **清理代码**（重构，但测试必须一直是绿的）
4. **循环往复**

如果 AI 先写了代码再补测试？这个 skill 会要求它删掉代码从头来。

### AI 遇到 bug 时

不许猜。必须按步骤排查：

1. **读完整的错误信息**（不许跳过）
2. **稳定复现**
3. **追踪数据流找根因**
4. **用最小改动验证假设**

如果连续 3 次修复都没用，AI 会停下来告诉你"这可能是架构问题"，而不是继续无限试。

### AI 说"搞定了"之前

必须拿出证据：

```
❌ "测试应该通过了"
❌ "看起来没问题"

✅ [运行 npm test] → 34/34 通过 → "所有测试通过"
```

没有运行验证命令就不许说完成。

### 大型任务

AI 会把计划拆成 2-5 分钟一步的小任务，每个任务派一个独立的子代理去执行，完成后自动做两轮审查（功能对不对 + 代码质量好不好），全部通过才算完。

## 包含的 Skill 一览

| Skill | 做什么 |
|-------|--------|
| Brainstorming | 头脑风暴 — 问问题、出方案、做设计 |
| Writing Plans | 把设计拆成 2-5 分钟一步的执行计划 |
| Test-Driven Development | 强制 TDD — 先测试后代码，不许反过来 |
| Systematic Debugging | 系统化排查 bug — 不许猜，必须找根因 |
| Verification Before Completion | 完成前必须拿证据 — 不许说"应该没问题" |
| Using Git Worktrees | 在隔离分支上工作 — 不污染主分支 |
| Executing Plans | 在新会话中按批次执行计划 |
| Subagent-Driven Development | 每个任务派独立子代理 + 双重审查 |
| Dispatching Parallel Agents | 多个独立问题并行处理 |
| Requesting Code Review | 完成后自动请求代码审查 |
| Receiving Code Review | 收到审查意见后的处理规则 |
| Finishing a Development Branch | 完成后选择合并/PR/保留/丢弃 |
| Writing Skills | 创建自己的 skill |

## 完整工作流是什么样的

假设你想给项目加一个用户认证功能：

```
你：我想加一个用户登录功能

AI：我先看看项目结构...你用的是什么认证方案？
    1. NextAuth.js（推荐）
    2. 自己写 JWT
    3. Firebase Auth
你：用 NextAuth

AI：（分段展示设计，逐个确认）
AI：计划已生成，共 5 个任务。用子代理模式执行。

[创建隔离分支]

任务 1：安装和配置 NextAuth
  [子代理按 TDD 实现]
  [审查通过] ✅

任务 2-5：同上...

AI：全部完成，测试 23/23 通过。
    1. 合并回 main
    2. 推送并创建 PR
    3. 保持现状
    4. 丢弃
你：2
AI：PR 已创建
```

## 与 yy-spec 的关系

| | Superpowers | yy-spec |
|---|---|---|
| 关注点 | 怎么写好代码 | 怎么理清需求 |
| 核心能力 | TDD、调试、审查 | 需求 → 设计 → 任务 → 实现 |
| 适合场景 | 任何编码任务 | 功能开发、Bug 修复 |

两者不冲突，可以同时安装。
